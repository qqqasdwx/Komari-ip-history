# 注入弹窗鉴权定稿

更新时间：2026-04-02

本文档用于固化 `Komari -> IPQ` 注入弹窗链路的权限、鉴权、回跳和游客可见性规则。

---

## 1. 总体目标

- 保留 `Komari` 页内弹窗体验。
- 不再把“跨站 iframe 里复用管理员 cookie”作为主方案。
- 管理员后台能力与注入弹窗能力分开设计。
- 游客链路允许只读，但必须受明确边界约束。

---

## 2. 角色边界

当前链路只区分两类身份：

- `Komari` 管理员
- `Komari` 游客

判断方式：

- 不依赖页面 DOM 猜测身份。
- 以 `Komari` 自身接口返回结果为准。
- 主题包变化不应影响身份判断。

---

## 3. 游客链路

### 3.1 是否允许游客查看

- 提供一个 **IPQ 游客只读全局开关**。
- 默认：**关闭**。
- 该开关只影响 **Komari 注入弹窗**。
- 不影响 IPQ 独立页面直链访问规则。

### 3.2 游客链路放行条件

只有同时满足以下条件，游客才允许打开只读弹窗：

1. 当前身份是 `Komari` 游客
2. IPQ 游客只读全局开关已开启
3. 当前节点在 `Komari` 中本身对游客可见

节点是否对游客可见，当前按 `Komari` 的公开节点规则跟随：

- 节点公开：`hidden = false`
- 节点隐藏：`hidden = true`

### 3.3 游客链路的交互

- 若游客只读全局开关关闭：
  - 不打开弹窗
  - 直接弹 `toast`
  - 文案暂不锁定，当前语义为：管理员未开放该功能
- 若开关开启且节点公开：
  - 直接打开只读弹窗
- 游客不会触发节点接入
- 游客不会进入 IPQ 管理后台

### 3.4 游客链路接口规则

- 游客详情接口采用**完全匿名只读**模式
- 不额外引入公开 token
- 单独拆公开接口，不复用后台详情接口
- 路由风格采用 `/public/...`

公开接口只返回**展示所需最小数据**，不返回：

- 节点 UUID
- 内部更新时间
- 报告配置
- 其他后台内部字段

### 3.5 游客链路里的 IP 展示

- IP 展示继续跟随 `Komari` 的游客公开策略
- 若 `Komari` 允许游客看部分 IP：
  - IPQ 也只显示脱敏 IP
- 若 `Komari` 不允许游客看 IP：
  - IPQ 保留 `IP` 这一项
  - 值显示 `N/A`

除了 IP 之外，当前结果里的其他内容允许完整展示，包括：

- 风险评分
- 风险因子
- 流媒体 / AI 服务解锁
- 邮局连通性与黑名单检测

---

## 4. 管理员链路

### 4.1 管理员查看规则

- 当前身份若为 `Komari` 管理员，则允许走管理员链路
- 管理员链路仍使用 IPQ 自身登录体系
- 不因为 `Komari` 已登录就直接跳过 IPQ 登录

### 4.2 已接入节点

- 若 IPQ 仍有有效登录态：
  - 直接打开完整详情
- 若 IPQ 没有有效登录态：
  - 跳转到 IPQ 独立登录页
  - 登录成功后继续执行原操作

### 4.3 未接入节点

- 只有 `Komari` 管理员可以触发接入
- 未接入节点不在 iframe 中登录或接入
- 统一跳到 IPQ 独立页面完成：
  - 登录
  - 接入
  - 回到结果查看

### 4.4 回跳策略

- 登录成功后继续执行鉴权前的原操作
- 接入成功后优先回到 `Komari` 弹窗详情
- 如果回到 `Komari` 弹窗失败：
  - 退化停留在 IPQ 独立详情页

### 4.5 登录态复用

- 管理员的 IPQ 登录态应尽量长期复用
- 只有在会话失效时才要求重新登录

---

## 5. 接口拆分原则

- **管理员接口** 与 **游客只读接口** 彻底分开
- 详情接口只负责**当前状态**
- 历史数据改走独立接口
- 不再把历史作为当前详情 payload 的一部分

---

## 6. 当前不采用的方案

以下方案当前不作为主实现：

- 在跨站 iframe 中长期复用管理员 cookie 作为主链路
- 在 header 中放管理员全权 token
- 让游客参与节点接入
- 依赖 DOM 结构猜测 `Komari` 管理员身份
- 公开接口与后台接口共用同一条详情路由再做字段裁剪

---

## 7. 当前实现结论

后续实现应按如下状态机推进：

1. 先通过 `Komari` 接口判断当前身份
2. 若为游客：
   - 游客只读关闭：toast
   - 游客只读开启且节点公开：匿名只读弹窗
3. 若为管理员：
   - 已接入且 IPQ 已登录：完整详情弹窗
   - 已接入但 IPQ 未登录：跳独立页登录，再继续原操作
   - 未接入：跳独立页登录并自动接入，完成后优先回到弹窗
