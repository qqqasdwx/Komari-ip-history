## 一、你刚刚指出的两个问题

### 1. Komari 详情页点击“打开/查看 IP 质量”后，反应慢，需要点多次

#### 旧逻辑

在 `internal/service/header.go` 的旧实现里：

```js
if (event.data.type === "open-standalone" && event.data.url) {
  closeModal();
  window.location.assign(event.data.url);
}
```

也就是说：
- 收到 `open-standalone` 事件后
- 直接用 **当前页面跳转** 到独立页
- 没有先尝试 `window.open`

#### 当前逻辑

当前本地改动变成了：

```js
if (event.data.type === "open-standalone" && event.data.url) {
  closeModal();
  var opened = window.open(event.data.url, "_blank", "noopener,noreferrer");
  if (!opened) {
    window.location.assign(event.data.url);
  }
}
```

也就是说现在会：
1. 先尝试 **新标签页打开**
2. 如果浏览器拦截或失败，再 fallback 到当前页跳转

#### 我当前判断

这条是**明确发生过交互语义变化**的，而且会直接影响用户点击反馈节奏。

潜在影响：
- 某些浏览器 / 某些主题 / 某些事件链下，`window.open()` 可能受到弹窗策略、焦点切换、异步事件时序影响
- 用户主观感受就会变成：**点了没反应、要再点几下**

#### 我的结论

- 这条需要你来定：
  - 如果你本来就是要“当前页立即跳转”，那现在的 `window.open + fallback` 就应视为**回归**
  - 如果你本来就是要“优先新标签页”，那就要继续排查为什么当前实现点击反馈慢

#### 相关文件

- 旧/当前对比文件：`internal/service/header.go`

### 2. 快照对比时间轴从“平滑拖动”变成“自动吸附时间节点”

#### 我查到的代码事实

我对比了：
- 当前工作区 `web/src/react-preview/App.tsx`
- `HEAD`
- 更早历史提交 `84b8236 refine history compare timeline and field metadata`

当前和 `HEAD` 中，时间轴核心逻辑都是：

```tsx
<input
  type="range"
  min={min}
  max={max}
  step={1000}
  value={props.leftTimestamp}
  onChange={(event) => props.onLeftChange(clampCompareTimestamp(Number(event.target.value), min, props.rightTimestamp))}
/>
```

并且最终对比对象使用：

```ts
pickHistoryEntryAtOrBefore(items, timestamp)
```

即：
- 滑块本身是按时间戳移动
- 但最终展示会选“**该时间点及之前最近的一条历史记录**”
- `step={1000}` 代表按秒级时间戳移动

#### 旧逻辑（我这批本地改动之前）

从**当前 `HEAD` 和更早历史提交对比看**：
- 我没有发现这批本地改动把时间轴从“平滑拖动”改成“节点吸附”的代码变化
- 也就是说，**至少不是我这轮本地 diff 直接把时间轴实现改坏的**

#### 当前逻辑

当前工作区里的时间轴实现和 `HEAD` 看到的是同一类逻辑：
- `range` 滑动
- `step=1000`
- `pickHistoryEntryAtOrBefore`

#### 我当前判断

如果你确认“之前是平滑拖动、不吸附”，那可能存在三种情况：

1. **真正的旧逻辑不在当前 `HEAD`，而在更早的某个历史版本**
2. **代码没变，但数据量 / 时间分布 / 样式层表现变了，导致主观体验像吸附**
3. **之前是刻意做过更丝滑的交互处理，但后来在某个更早提交中已经退化了**

#### 我的结论

- 就“我这批本地页面改动”来看，**没有直接证据表明是我把快照对比时间轴改成吸附的**
- 但你指出的问题是有效的，值得继续往更早历史里追根

#### 相关文件

- `web/src/react-preview/App.tsx`
