# 发版说明

本项目采用两条发布线：

- `latest`
  用于开发联调和快速试用
- `v0.y.z`
  用于正式版本

## 版本规则

当前按务实版 SemVer 执行。

### PATCH

用于：

- bug 修复
- 安全修复
- 样式修正
- 兼容性修正
- 不改变主要使用方式的小改动

### MINOR

用于：

- 新增用户可见功能
- 接入流程变化
- 历史、快照、注入体验的结构性变化
- API 结构变化
- 有升级影响的数据模型变更

### MAJOR

用于：

- 不兼容的 API 或行为变更
- 需要明确升级迁移说明的大版本发布

当前首个稳定版本采用 `v1.0.0`。

## 当前发布产物

当前以 Docker 镜像为主：

- `ghcr.io/qqqasdwx/komari-ip-history:latest`
- `ghcr.io/qqqasdwx/komari-ip-history:v0.y.z`

正式部署建议固定到具体版本，不要长期跟随 `latest`。

## 安装脚本来源

节点设置页生成的接入命令会根据当前运行版本选择安装脚本来源：

- 固定版本镜像，例如 `v0.1.0`，使用同名 tag 下的 `deploy/install.sh`
- `latest` 镜像使用默认分支下的 `deploy/install.sh`
- 开发环境默认使用默认分支下的 `deploy/install.sh`

如果需要临时指定脚本来源，可以通过环境变量覆盖：

- `IPQ_VERSION`：当前运行版本
- `IPQ_COMMIT`：当前构建提交
- `IPQ_INSTALLER_SCRIPT_REF`：指定 Git ref
- `IPQ_INSTALLER_SCRIPT_URL`：指定完整安装脚本 URL

固定版本部署时，服务端运行版本、页面接入命令和安装配置接口会返回一致的脚本来源。

## 正式发版流程

推荐流程：

1. 在本地完成验证
2. 确认 `master` 已包含本次发布代码
3. 创建 annotated tag
4. 推送 tag
5. 等待 GitHub Actions 构建并发布对应镜像
6. 创建同名 GitHub Release

示例命令：

```bash
git checkout master
git pull --ff-only origin master
git tag -a v0.1.0 -m "Release v0.1.0"
git push origin v0.1.0
```

## GitHub Release

建议 Release 内容包含：

- 本版主要变更
- 升级注意事项
- 已知限制

可以在 GitHub 自动生成的 release notes 基础上补充说明。

## 当前不做的事

当前不包含：

- 自动跑 E2E 的 GitHub Actions 发布门禁
- 多平台二进制附件
- 复杂的 `beta` / `rc` 发布体系
- 独立维护发布分支
