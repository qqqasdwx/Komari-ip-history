# 集成与行为说明

本文档面向部署者和贡献者，说明项目当前稳定的集成方式和主要功能行为。

## 项目定位

`Komari-ip-history` 是一个独立部署的 IP 质量服务，用来把节点的 IP 质量结果接入 `Komari`。

它不修改 `Komari` 官方源码，而是通过 `Komari` 的 `custom_head` 注入一个入口按钮，再由本服务提供：

- 后台管理页面
- 节点结果页面
- 历史变化页面
- 快照对比页面
- 节点侧上报配置和安装脚本

## 默认集成方式

当前默认按下面的方式设计：

- IPQ 作为独立服务部署
- `Komari` 通过 `custom_head` 注入入口
- 注入脚本只负责桥接、识别节点、挂载按钮和跳转/弹窗
- 复杂展示和业务逻辑都在 IPQ 自身页面完成

当前不把下面这些前提当成默认要求：

- 与 `Komari` 同域
- 与 `Komari` 同机
- 默认走 `/ipq/` 子路径

## 查看流程

### 管理员

`Komari` 管理员点击入口后：

- 如果节点已接入且 IPQ 已有登录态，优先直接打开页内弹窗
- 如果节点未接入，或 IPQ 未登录，则跳转到 IPQ 独立页面完成登录或接入
- 完成后优先回到 `Komari` 页内继续查看

### 游客

游客链路受全局“游客只读”开关控制。

- 默认关闭
- 开启后，游客可以查看当前节点的匿名只读结果
- 游客不能接入节点，也不能进入管理后台

## 节点与目标 IP

一个 `Komari` 节点下可以配置多个目标 IP。

当前行为：

- 每个目标 IP 单独保存当前结果
- 每个目标 IP 单独累积历史快照
- 结果页、历史页、快照对比页都支持按目标 IP 查看
- 节点侧安装命令会按当前目标 IP 顺序依次探测并上报

## 历史与快照

历史能力拆成两个页面：

- 历史主页面
- 快照对比页面

### 历史主页面

历史主页面的主视角是“字段变化列表”，不是“快照列表”。

支持：

- 按时间范围筛选
- 按目标 IP 筛选
- 按字段筛选
- 分页

### 快照对比页面

快照对比页面用于比较同一目标 IP 的两个历史时间点。

当前行为：

- 顶部使用双游标时间轴选择两个时间点
- 页面下方并排展示两个完整快照
- 变化字段会高亮
- 支持收藏单条历史快照

## 历史保留策略

历史继续按完整快照存储，不改成 diff-only。

当前保留策略：

- 全局配置项：`retention_days`
- `-1` 表示永久保留
- `1` 及以上表示保留最近 N 天
- 已收藏快照不会被自动清理
- 取消收藏后重新受全局保留策略约束

保留策略只作用于历史快照，不影响：

- 节点本身
- 目标 IP 本身
- 当前结果

## 公开访问地址

项目默认按当前访问地址生成：

- 注入脚本地址
- 页面跳转地址
- 节点上报地址

如果你的部署前面有反向代理，应正确传递：

- `Host`
- `X-Forwarded-Proto`

如果你希望统一使用固定外部地址，可设置：

- `IPQ_PUBLIC_BASE_URL`

## 主题兼容

当前主要保证：

- 默认主题下可正常注入和查看
- PurCarte 主题下可正常注入和查看

主题适配属于体验增强，不改变项目的主集成模型。
