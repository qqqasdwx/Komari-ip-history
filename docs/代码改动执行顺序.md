# 代码改动执行顺序

本文档是在 `docs/代码影响清单.md` 基础上，进一步整理出的**建议实施顺序**。

目标不是一次性把所有旧实现全推翻，而是按“先收口鉴权主链路，再做接口减法，最后再清理遗留”的顺序推进。

---

## 1. 总体原则

当前代码改动建议遵守这 5 条原则：

1. **先改权限边界，再改 UI 细节**
2. **先收口游客 / 管理员链路，再删旧功能**
3. **先拆接口边界，再谈缓存和性能**
4. **先让主路径成立，再清 legacy**
5. **最后再统一改 E2E / CI**

---

## 2. 推荐执行顺序总览

建议按下面 6 个阶段推进：

### 阶段 A：维持 React 主线

这一步已经完成，不再回头。

### 阶段 B：收口注入弹窗鉴权主链路

这一步已完成，当前代码已经具备：

- `Komari` 接口身份判断
- 游客匿名只读
- 管理员独立登录 / 接入 / 回跳

### 阶段 C：拆详情接口与历史接口

这一步也已完成：

- 详情只返回当前状态
- 历史走独立接口

### 阶段 D：按新状态机收口 loader / `connect`

这一步也已完成，当前主链路已改成：

- 游客匿名只读
- 管理员独立页登录 / 接入后回到弹窗

### 阶段 E：继续清 legacy 与旧变化线

这一步已经做掉了第一轮：

- 前端入口不再保留 legacy 分支
- `legacy-app.ts` 已删除
- 变化规则接口和旧变化工具文件已删除
- 旧的 Playwright 脚本已清理

### 阶段 F：最后再补 Playwright / CI

Playwright / CI 这一阶段已经做完第一轮接入，后续只需要继续补覆盖面。

## 3. 阶段 A：React 主线

### 当前执行状态

- 已拍板：**React 为唯一前端主线**
- 已完成：**默认前端入口切到 React**
- legacy 当前仅保留为兼容残留，不再是默认实现

这一步当前不再是阻塞项。

---

## 4. 阶段 B：收口注入弹窗鉴权主链路

状态：已完成

## 目标

把当前产品真正最关键的主链路收稳：

1. 本服务生成 header
2. 用户复制到 Komari
3. Komari 节点详情页出现按钮
4. 点击后先通过 `Komari` 接口判断当前身份
5. 再进入：
   - 游客只读链路
   - 管理员完整链路
6. 用户看到当前 IP 质量结果

### B1. 明确 Header 预览的地址策略

核心目标：

- Header 默认优先基于 `IPQ_PUBLIC_BASE_URL` 生成
- 不再继续把“当前站点 origin + /ipq”当成默认产品假设

### B2. 接入 `Komari` 身份判断

核心目标：

- 不靠 DOM 判断管理员身份
- 统一通过 `Komari` 接口判断当前是：
  - 游客
  - 管理员

### B3. 拆游客只读接口

核心目标：

- 单独增加 `/public/...` 公开只读接口
- 不复用管理员详情接口做字段裁剪
- 游客只读默认关闭
- 游客只读开启时，只返回当前结果所需最小数据

### B4. 改管理员登录 / 接入 / 回跳链路

核心目标：

- 不再把“跨站 iframe 内管理员登录闭环”当主方案
- 管理员未登录时跳独立页
- 登录后继续原操作
- 未接入时自动接入
- 完成后优先回到 `Komari` 弹窗

### B5. 增加游客关闭态提示

核心目标：

- 游客只读关闭时
- 不打开弹窗
- 直接 toast 提示“管理员未开放该功能”

### 本阶段验收结果

1. 游客只读关闭时，游客点击只弹 toast
2. 游客只读开启且节点公开时，游客可打开匿名只读弹窗
3. 管理员未登录 IPQ 时，会跳独立页登录并继续原操作
4. 管理员未接入节点时，会跳独立页完成接入并优先回到弹窗

---

## 5. 阶段 C：拆详情接口与历史接口

状态：已完成

## 目标

把“当前状态”和“历史数据”彻底分开。

### C1. 管理员详情接口只返回当前状态

### C2. 游客只读详情接口只返回最小展示数据

### C3. 历史数据改走独立接口

### 本阶段验收结果

1. 详情接口默认不再背历史 payload
2. 游客只读接口和管理员接口边界已经拆开
3. 历史已改走独立接口

---

## 6. 阶段 D：收口 loader / `connect`

状态：已完成

## 目标

把现有 loader 和 `connect` 页改造成符合新状态机的实现，而不是继续承载旧主链路。

### D1. loader 负责分流，不负责复杂鉴权

### D2. `connect` 页降级为管理员登录 / 接入 / 回跳过渡页

### D3. 保留 iframe 弹窗，但不在其中完成高风险登录动作

### 本阶段验收结果

1. loader 分流逻辑和产品文档一致
2. `connect` 页已降级为管理员过渡页
3. 页面状态回跳可用

---

## 目标

在主链路稳定后，再继续清：

- legacy 前端残留
- history / changes / 变化规则旧线
- 旧文档中的误导性主叙事

当前状态：

- `loader` 首轮性能收口已完成，不再是这里的主任务
- 这一阶段已经完成第一轮减法
- 下一步更应该继续看：
  - 历史接口是否还要进一步收口
  - 不再使用的历史叙事代码

---

## 8. 阶段 F：最后再补验证体系

## 目标

围绕新主链路补 Playwright / CI：

- 游客只读开启 / 关闭
- 管理员未登录回跳
- 管理员未接入自动接入
- 详情 / 历史接口分离后的回归

当前状态：

- 已新增 `web/playwright/verify-embed-auth-flows.mjs`
- `deploy/dev/workspace/run-e2e.sh` 已切到当前有效脚本组合
- 新鉴权脚本已支持自清理，不再把测试节点和游客开关留在环境里
- 已手工跑通：
  - 游客关闭 -> toast
  - 游客开启 -> 匿名只读弹窗
  - 管理员已登录 -> 完整详情弹窗
  - 管理员未登录 -> 独立登录后回弹窗
  - 管理员未接入 -> 自动接入后回弹窗
- 下一步是继续补更多回归覆盖，而不是再改入口
