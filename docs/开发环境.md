# 开发环境

本项目推荐使用两种开发方式：

- 日常开发：本机直接运行 Go 后端和前端开发服务器
- 联调与回归：使用 Docker Compose 启动完整环境

## 日常开发

适用场景：

- 修改 Go 后端
- 修改前端页面和样式
- 跑本地构建和单元测试

推荐方式：

- 后端：本机直接运行
- 前端：本机直接运行 Vite
- 数据库：本地 SQLite 文件

建议关注这些环境变量：

- `IPQ_APP_ENV`
- `IPQ_LISTEN`
- `IPQ_DB_PATH`
- `IPQ_PUBLIC_BASE_URL`
- `IPQ_BASE_PATH`
- `VITE_BASE_PATH`

默认推荐：

- `IPQ_BASE_PATH=""`
- `VITE_BASE_PATH=/`

## 联调环境

适用场景：

- 与 `Komari` 联调
- 验证注入链路
- 跑 Playwright
- 做回归验证

入口文件：

- `compose.dev.yml`

联调环境中的主要服务：

- `komari-default`
- `komari-purcarte`
- `workspace`
- `proxy`

默认访问入口：

- IPQ 后端：`http://127.0.0.1:8090`
- 前端开发服务器：`http://127.0.0.1:5173`
- Komari 默认主题：`http://127.0.0.1:8080`
- Komari PurCarte 主题：`http://127.0.0.1:8081`

容器内 Playwright 回归会通过 Caddy 访问：

- Komari 默认主题：`http://proxy:8080`
- Komari PurCarte 主题：`http://proxy:8081`
- IPQ 注入地址：`http://proxy:8090`

`proxy:8090` 反向代理到 `workspace:8090`。这样 e2e 中 Komari 和 IPQ 都位于 `proxy` 主机下，只是端口不同，能更接近浏览器真实访问时的 iframe 登录和 cookie 行为。

启动完整开发环境并重建测试数据：

```bash
docker compose -f compose.dev.yml up -d --build
docker compose -f compose.dev.yml exec -T workspace sh /workspace/deploy/dev/workspace/prepare-dev-env.sh
```

这个脚本会安装依赖、启动 Go 后端、启动 Vite 前端、等待 Komari 可用，并重建开发种子节点。

联调环境会启动两个 Komari 实例：`8080` 保持默认主题，`8081` 会自动下载并启用 PurCarte 主题。首次准备 PurCarte 需要能访问 GitHub release；后续会复用已安装主题。Playwright 嵌入流程会分别覆盖这两个实例。

## 回归测试

完整 e2e 回归命令：

```bash
docker compose -f compose.dev.yml exec -T workspace sh /workspace/deploy/dev/workspace/run-e2e.sh
```

这个脚本会：

- 重建 IPQ 前端并重启后端
- 重建开发种子节点
- 确保 PurCarte 主题启用
- 运行真实用户 onboarding 流程：从 Komari UI 创建节点开始，配置 IPQ Header，点击注入入口接入节点，配置多目标 IP，并验证详情、历史和快照对比
- 运行节点页回归
- 分别运行默认主题和 PurCarte 主题下的嵌入权限回归

真实用户 onboarding 脚本位于 `web/playwright/verify-real-user-onboarding.mjs`。它会创建临时 `Playwright Real User *` 节点，并在结束时清理临时 Komari/IPQ 节点。

## 开发种子节点

如果需要快速重建一组固定联调节点，在仓库根目录执行：

```bash
docker compose -f compose.dev.yml exec -T workspace sh /workspace/deploy/dev/workspace/seed-dev-nodes.sh
```

如果需要临时验证真正的节点列表空状态，可以只清理开发种子节点：

```bash
docker compose -f compose.dev.yml exec -T workspace sh /workspace/deploy/dev/workspace/seed-dev-nodes.sh --cleanup-only
```

验证后重新执行默认种子命令即可恢复完整测试数据。

脚本会重建以下开发节点：

- `开发种子-空节点`
- `开发种子-有目标无数据`
- `开发种子-单IP无历史`
- `开发种子-单IP历史`
- `开发种子-多IP历史`
- `开发种子-多快照对比`
- `开发种子-长名称与密集数据节点-*`

这些种子数据用于覆盖无目标、有目标无数据、有当前结果但无历史、多条历史、多条快照、收藏快照、多目标切换和移动端密集内容布局。

## 前端修改后页面没更新

如果前端改了但页面没有变化，优先执行：

```bash
docker compose -f compose.dev.yml exec -T workspace sh /workspace/deploy/dev/workspace/reload-ipq.sh
```

这个脚本会：

- 重新构建前端
- 重启工作容器中的 Go 后端
- 等待健康检查恢复

建议排查顺序：

1. 先运行 `reload-ipq.sh`
2. 再检查页面 DOM、截图和接口返回
3. 不要先把问题归因到浏览器缓存
