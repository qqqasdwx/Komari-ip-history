# 开发环境

本文档描述当前阶段 1 的本地开发方式。

## 1. 目标

开发环境默认采用单一入口反代：

- `Komari` 挂在根路径 `/`
- 本服务挂在子路径 `/ipq/`
- 浏览器只访问一个入口地址，避免同源和子路径问题在开发阶段被掩盖
- 本项目自己的开发工具链统一放进一个开发工作容器里，不在宿主机本地执行 `go build`、`go test`、`npm install`、`npm run build`

## 2. 容器结构

当前开发环境包含三个服务：

- `proxy`：统一入口反代
- `komari`：官方 Komari 容器
- `workspace`：本项目开发工作容器，内置 Go 和 Node.js 依赖，整仓代码挂载到 `/workspace`

其中：

- `workspace` 固定容器名为 `ipq-workspace-dev`
- 本项目的后端开发进程监听 `8090`
- 本项目的前端 Vite 开发进程监听 `5173`
- 这两个进程都通过 `docker exec` 在容器内启动

## 3. 启动基础环境

在仓库根目录执行：

```powershell
docker compose -f compose.dev.yml up -d --build
```

启动后会得到：

- `workspace` 开发工作容器
- `Komari` 容器
- `Caddy` 统一入口

注意：此时本项目的 Go 后端和 Vite 前端进程还没有启动，需要在下一步通过 `docker exec` 启动。

## 4. 初始化项目依赖

所有依赖安装都在 `workspace` 容器中完成。

推荐直接执行仓库内脚本：

```powershell
docker exec -it ipq-workspace-dev sh /workspace/deploy/dev/workspace/bootstrap.sh
```

它会在容器内完成：

- `go mod download`
- `cd /workspace/web && npm install`

## 5. 启动开发进程

启动 Go 后端：

```powershell
docker exec -it ipq-workspace-dev sh /workspace/deploy/dev/workspace/start-backend.sh
```

启动前端 Vite 开发服务器：

```powershell
docker exec -it ipq-workspace-dev sh /workspace/deploy/dev/workspace/start-frontend.sh
```

启动后默认访问：

- `http://localhost:8080/`：Komari
- `http://localhost:8080/ipq/`：本服务独立前端

## 6. 停止与查看日志

停止 Go 后端：

```powershell
docker exec -it ipq-workspace-dev sh /workspace/deploy/dev/workspace/stop-backend.sh
```

停止前端开发服务器：

```powershell
docker exec -it ipq-workspace-dev sh /workspace/deploy/dev/workspace/stop-frontend.sh
```

查看 Go 后端日志：

```powershell
docker exec -it ipq-workspace-dev sh -lc "tail -f /tmp/ipq-backend.log"
```

查看前端日志：

```powershell
docker exec -it ipq-workspace-dev sh -lc "tail -f /tmp/ipq-frontend.log"
```

## 7. 在容器内执行构建与校验

不要在宿主机本地构建，统一在 `workspace` 容器内执行。

前端构建：

```powershell
docker exec -it ipq-workspace-dev sh -lc "cd /workspace/web && npm run build"
```

后端构建：

```powershell
docker exec -it ipq-workspace-dev sh -lc "cd /workspace && go build ./cmd/ipq"
```

后端测试：

```powershell
docker exec -it ipq-workspace-dev sh -lc "cd /workspace && go test ./..."
```

## 8. 默认账号

开发环境默认初始化为：

- 本服务：admin / admin
- Komari：admin / admin

## 9. 持久化目录与缓存

当前默认持久化目录：

- `./data/komari`：Komari 数据
- `./data/ipq`：本服务 SQLite 数据

当前默认容器卷：

- `go-mod-cache`：Go 模块缓存
- `go-build-cache`：Go 构建缓存
- `npm-cache`：npm 缓存
- `web-node-modules`：前端 `node_modules`

## 10. 环境变量

可用环境变量见仓库根目录 `.env.example`，当前核心项包括：

- `IPQ_APP_ENV`：运行模式，`production` 或 `development`
- `IPQ_LISTEN`：Go 服务监听地址
- `IPQ_BASE_PATH`：服务挂载路径前缀，默认 `/ipq`
- `IPQ_DB_PATH`：SQLite 文件路径
- `IPQ_SESSION_COOKIE`：登录会话 Cookie 名称
- `IPQ_DEFAULT_ADMIN_USERNAME`：默认管理员用户名
- `IPQ_DEFAULT_ADMIN_PASSWORD`：默认管理员密码
- `IPQ_PUBLIC_BASE_URL`：未来生产环境可选的外部访问基地址
- `IPQ_COOKIE_SECURE`：是否使用 secure cookie
- `VITE_BASE_PATH`：前端开发和构建时使用的子路径前缀
