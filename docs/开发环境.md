# 开发环境

本文档描述当前推荐的开发方式。

> 当前推荐采用“双轨制”：  
> **Debian 原生日常开发** + **Docker 联调 / E2E / CI**。  
> 相关讨论见 `docs/Debian原生开发环境迁移方案.md`。

---

## 1. 当前结论

不再把 Docker 当成唯一开发方式。

当前推荐分工如下：

### 日常开发

- 优先在 Debian 上原生运行
- 适合改：
  - Go 后端
  - 前端页面
  - 样式
  - 接口
  - 测试

### 联调 / 回归 / 演示

- 使用 Docker Compose
- 适合验证：
  - Komari 联调
  - 注入链路
  - 完整浏览器流程
  - E2E
  - CI

---

## 2. Debian 原生日常开发

### 2.1 适用场景

适合：

- 快速改代码
- 调页面
- 跑本地构建
- 跑单元测试

### 2.2 当前建议

- 后端：本机直接运行
- 前端：本机直接运行 Vite
- 数据库：本地 SQLite 文件

### 2.3 当前需要关注的配置

建议关注这些环境变量：

- `IPQ_APP_ENV`
- `IPQ_LISTEN`
- `IPQ_DB_PATH`
- `IPQ_PUBLIC_BASE_URL`
- `VITE_BASE_PATH`

其中：

- `IPQ_PUBLIC_BASE_URL` 用于生成对外可访问的 header 与跳转地址
- 当前默认推荐：
  - `IPQ_BASE_PATH=""`
  - `VITE_BASE_PATH=/`
- `VITE_BASE_PATH` / `IPQ_BASE_PATH` 仍可保留，用于兼容子路径场景，但不再是默认前提

---

## 3. Docker 联调环境

### 3.1 角色

Docker 继续保留，主要用于：

- 启动 Komari
- 启动统一入口或联调环境
- 跑 Playwright
- 跑 CI
- 演示完整链路

### 3.2 当前 compose

当前仓库中已有：

- `compose.dev.yml`

它仍然是现阶段的主要联调环境入口。

### 3.3 当前定位

Docker 环境现在是：

- **联调环境**
- **验证环境**
- **CI 环境**

而不是唯一开发入口。

---

## 4. 当前 Docker 环境的大致结构

当前联调环境主要包含：

- `proxy`
- `komari`
- `workspace`

其中：

- `komari` 用于联调 `Komari`
- `workspace` 用于统一执行构建、前端验证和 E2E
- `proxy` 用于现有联调场景下的统一入口

当前 compose 默认还会直接暴露：

- `8090`：本服务后端默认独立入口
- `5173`：前端 Vite 开发端口

需要注意：

- 这套“统一入口 + 子路径”联调方式现在仍然有价值
- 但它只是一种**联调模型**
- 不再代表产品默认部署形态

当前默认独立部署入口应理解为：

- 页面：`http://host:port/#/...`
- API：`http://host:port/api/v1/...`
- loader：`http://host:port/embed/loader.js`

---

## 5. 当前推荐实践

### 5.1 日常改代码

优先使用 Debian 原生环境。

### 5.2 需要验证 Komari 注入时

使用 Docker Compose 拉起联调环境。

### 5.3 需要跑回归或 CI 时

继续走 Docker 环境。

---

## 6. 当前保留的关键能力

当前开发环境仍应保证这些能力可用：

- 本服务构建
- 本服务测试
- 前端构建
- Playwright 注入验证
- Playwright 主流程验证
- Komari 联调

---

## 7. 默认账号

开发环境默认账号仍为：

- 本服务：`admin / admin`
- Komari：`admin / admin`

---

## 8. 数据目录

当前仍建议保留以下目录语义：

- `./data/ipq`：本服务 SQLite 数据
- `./data/komari`：Komari 数据

这样无论是原生模式还是 Docker 模式，目录含义都更一致。

---

## 9. 开发种子节点

如果需要快速重建一组固定联调节点，可以在仓库根目录执行：

```bash
docker compose -f compose.dev.yml exec -T workspace sh /workspace/deploy/dev/workspace/seed-dev-nodes.sh
```

脚本会清理旧的 `开发种子-*` 节点，并重建：

- `开发种子-空节点`
  - 无目标 IP，用于验证空状态
- `开发种子-单IP历史`
  - 单 IP + 3 条历史，用于历史页和对比
- `开发种子-多IP历史`
  - 多 IP + 分 IP 历史，用于标签切换和多 IP 上报

这个脚本只用于开发环境，不是生产功能。
