# 文档索引

这份索引用于说明当前 `docs/` 目录里哪些文档是**当前有效**的，哪些文档主要是**历史记录 / 过程文档**。

---

## 一、当前优先阅读

如果要快速了解项目当前方向，建议按这个顺序读：

1. `docs/项目背景.md`
   - 项目是什么
   - 当前产品目标是什么

2. `docs/Komari集成方式重定稿.md`
   - 当前默认按什么集成方式设计
   - 为什么不再把同域 `/ipq/` 子路径当默认前提
   - 为什么仍然优先争取页内弹窗

3. `docs/注入弹窗鉴权定稿.md`
   - Komari 注入弹窗的游客/管理员权限边界
   - 公开只读、管理员登录、回跳与接入的状态规则

4. `docs/已确认事项.md`
   - 当前已经拍板的产品与技术结论

5. `docs/待确认事项.md`
   - 当前还没最终拍板的问题

6. `docs/开发环境.md`
   - 当前推荐的开发方式
   - Debian 原生开发与 Docker 联调如何分工

7. `docs/当前工作进度.md`
   - 代码库当前大致处于什么状态
   - 现有实现和当前方向之间有哪些偏差

8. `docs/任务清单-多IP与历史改造.md`
   - 节点下多目标 IP 的实现边界
   - 每 IP 当前结果与历史的落地进度

9. `docs/任务清单-历史视图增强.md`
   - 历史独立页面与快照对比的落地进度

10. `docs/代码影响清单.md`
   - 当前方向会影响到哪些代码模块

11. `docs/代码改动执行顺序.md`
   - 如果要开始改代码，推荐先做什么、后做什么

12. `docs/前端技术选型定稿.md`
   - 当前前端技术栈最终应参考什么
   - 哪些参考 `komari-web`
   - 哪些不需要照搬

## 三、当前文档口径

从现在开始，文档口径以这几条为准：

- 本服务是**独立服务**
- 默认按**跨域独立部署**设计
- Header 只做**桥接和入口注入**
- 产品体验上仍然**优先争取页内弹窗**
- 工程上必须保留**独立页面兜底**
- 注入弹窗鉴权以 `docs/注入弹窗鉴权定稿.md` 为准
- 节点下支持多个手动维护的目标 IP
- 当前展示重点包括：
  - 当前 IP 质量结果
  - 按目标 IP 查看历史
- “变化规则 / 趋势分析”不再作为当前主产品方向

## 四、后续整理原则

后续新增或修改文档时，建议遵守：

1. 先更新 `已确认事项.md`
2. 如果涉及架构级纠偏，再同步更新 `Komari集成方式重定稿.md`
3. 如果只是实施层变化，再更新 `开发环境.md` 或 `当前工作进度.md`
4. 历史文档尽量不再继续堆新结论，避免新旧混杂
