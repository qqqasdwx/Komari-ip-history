# 阶段 1 现状

本文档用于沉淀阶段 1 已实现内容、最近一次功能提交做了什么，以及对应的验收结果。

## 1. 当前阶段 1 范围

- 独立 Go 服务，挂载在 `/ipq/` 子路径下。
- 独立 Web 前端，负责登录、节点列表、节点详情、历史页、系统配置。
- 通过 `Komari` 自定义 header 注入少量 JavaScript，不修改 `Komari` 官方源码。
- 在 `Komari` 节点详情页注入节点级按钮，由用户主动把节点接入本服务。
- 注入端只负责按钮、弹层和桥接；详细展示由本服务页面承载。
- 本服务使用 `SQLite` 存储管理员、节点、当前结果、历史记录和展示配置。
- 阶段 1 使用开发模式 mock 数据打通主链路，并已补齐节点级 reporter token 上报接口与展示链路。

## 2. 已完成主链路

- 管理员登录与会话：
  - 单管理员模式
  - 默认账号 `admin/admin`

- 节点接入：
  - `Komari` 节点详情页显示 `添加 IP 质量检测` / `查看 IP 质量`
  - 点击后按 `uuid` 幂等注册节点
- 独立前端：

  - 系统配置页
- 系统配置：

  - 全局字段开关
  - 管理员改名改密
- 开发环境：
  - `docker compose` 启动 `proxy + komari + workspace`
  - `go build`、`npm run build`、Playwright 验证均在容器内执行

## 15. 本次功能更新

本次更新聚焦“GitHub Actions CI 收口”。

- 新增统一 CI 工作流 `.github/workflows/ci.yml`。
- CI 目前拆成两个 job：
  - `build`
  - `e2e`

  - `go test ./...`
  - `go build ./cmd/ipq`

- `e2e` 直接复用当前开发环境：
  - `docker compose -f compose.dev.yml up -d --build`
  - `bootstrap.sh`
  - `start-backend.sh`
  - `start-frontend.sh`
  - `run-e2e.sh`
- 开发工作容器镜像现在内置 Playwright Chromium 与系统依赖，避免 CI 里临时找浏览器。
- 新增 `run-e2e.sh`，用于统一在容器内执行：
  - `verify-injection.mjs`
  - `verify-stage1-flows.mjs`
- `bootstrap.sh` 已切换为 `npm ci`，与 CI 保持一致。
- CI 失败时会收集：
  - `playwright-output`
  - `ipq-backend.log`
  - `ipq-frontend.log`
  - `docker compose logs`
- 额外修复了一个容器内前端构建稳定性问题：
  - `npm run build` 前会先清理 `node_modules/.vite-temp`

## 21. 本次功能更新

本次更新聚焦“节点 reporter token 与真实上报接口”。

- 节点模型新增每节点独立 `ReporterToken`。
- 节点注册时会自动生成 reporter token；历史老节点如果缺少 token，会在再次接入时自动补齐。
- 后端新增公开上报接口：
  - `POST /api/v1/report/nodes/:uuid`
  - 支持 `X-IPQ-Reporter-Token`
  - 兼容 `Authorization: Bearer ...`
- 上报请求体当前支持：
  - `result`：必填
  - `summary`：可选
  - `recorded_at`：可选
- 每次成功上报都会同时更新：
  - 节点当前结果
  - 当前摘要
  - 最近更新时间
  - 历史记录
- 管理端节点详情页的“节点接入配置”区块已从占位说明升级为真实配置区，提供：
  - 浏览器当前地址下的完整上报 URL
  - 开发环境 compose 网络地址下的完整上报 URL
  - reporter token 复制
  - reporter token 重置
  - `curl` 示例复制
- `/auth/me` 会返回 `public_base_url`，用于前端优先展示生产环境外部访问地址。
- Playwright 主流程验证已补充 reporter 上报链路：
  - 从详情接口读取 reporter token
  - 调用上报接口写入新结果
  - 刷新详情页确认摘要更新
  - 校验历史条数增加
