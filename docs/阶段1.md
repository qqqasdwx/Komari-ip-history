# 阶段 1 目标

1. **服务端**
   - 通过 API 获取 Komari 节点列表（通过 Komari 官方接口/SDK）
   - 提供 API 获取单节点状态（初期可以返回模拟数据）

2. **前端**
   - 在 Komari 节点详情页注入一个按钮（“查看 IP 质量”）
   - 点击按钮时，调用服务端 API 获取节点状态
   - 弹窗显示节点状态数据（初期可用模拟 JSON）

3. **技术栈**
   - Go，与 Komari 官方保持一致
   - 阶段 1 不涉及节点端 IPQuality 上报
   - 数据暂时可存内存或轻量缓存，不使用数据库

4. **Docker**
   - 本地运行 Docker 版 Komari 供开发和调试

---

# 阶段性验收标准（阶段 1）

- 服务端能成功调用 Komari API 获取节点列表
- 节点详情页显示按钮
- 点击按钮能请求服务端接口并返回对应节点状态
- 前端弹窗显示模拟数据，数据与节点 UUID 对应
- 技术栈与 Komari 官方保持一致，便于后续集成和迭代
