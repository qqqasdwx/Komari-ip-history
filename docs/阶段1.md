# 阶段 1 现状

本文档用于沉淀阶段 1 已实现内容、最近一次功能提交做了什么，以及对应的验收结果。

## 1. 当前阶段 1 范围

- 独立 Go 服务，挂载在 `/ipq/` 子路径下。
- 独立 Web 前端，负责登录、节点列表、节点详情、历史页、系统配置。
- 通过 `Komari` 自定义 header 注入少量 JavaScript，不修改 `Komari` 官方源码。
- 在 `Komari` 节点详情页注入节点级按钮，由用户主动把节点接入本服务。
- 注入端只负责按钮、弹层和桥接；详细展示由本服务页面承载。
- 本服务使用 `SQLite` 存储管理员、节点、当前结果、历史记录和展示配置。
- 阶段 1 使用开发模式 mock 数据打通主链路，真实节点上报不在当前范围内。

## 2. 已完成主链路

- 管理员登录与会话：
  - 单管理员模式
  - 默认账号 `admin/admin`
  - 服务端 session + HttpOnly cookie
- 节点接入：
  - `Komari` 节点详情页显示 `添加 IP 质量检测` / `查看 IP 质量`
  - 点击后按 `uuid` 幂等注册节点
- 独立前端：
  - 节点列表页
  - 节点详情页
  - 历史页
  - 系统配置页
- 系统配置：
  - header 预览
  - 复制短 loader 版 / 完整内联版
  - 全局字段开关
  - 管理员改名改密
- 开发环境：
  - `docker compose` 启动 `proxy + komari + workspace`
  - `go build`、`npm run build`、Playwright 验证均在容器内执行

## 3. 本次提交内容

本次提交聚焦“当前状态总览”的结构化展示。

- 节点详情页和 `embed=1` 弹窗视图不再默认整块展示 JSON。
- 当前结果会优先按以下分组做结构化渲染：
  - `Meta`
  - `Score`
  - `Media`
  - `Mail`
- 未被上述分组消费的动态字段，继续显示在“其他字段”JSON 兜底区。
- 结构化展示仍然基于过滤后的结果渲染，不绕过全局字段开关。
- 补充了对应样式，使结构化展示在独立页和嵌入页保持一致。
- 补强 Playwright 验证脚本，明确校验详情页和嵌入页是否出现结构化分组。
- 将容器内 `go build ./cmd/ipq` 生成的仓库根目录二进制 `ipq` 纳入 `.gitignore`，避免污染工作区。

## 6. 本次功能更新

本次更新聚焦“节点列表总览可读性”。

- 节点列表接口现在会返回当前结果 JSON，供前端做轻量摘要渲染。
- 节点列表页不再只显示一行 `current_summary` 文本。
- 有数据节点会按三个固定区块展示摘要：
  - 风险概览
  - 媒体能力
  - 邮件能力
- 摘要渲染复用当前结果过滤逻辑，因此不会绕过全局字段开关。
- 无数据节点会明确显示 `N/A` 和“等待首份检测结果”。
- Playwright 主流程验证补充了列表页检查：
  - 至少有一张结构化摘要卡片
  - 至少有一张无数据卡片
  - 从列表点击进入详情页后，详情页结构化展示仍正常

## 9. 本次功能更新

本次更新聚焦“历史页结构化展示与上一条对比”。

- 历史页详情不再默认只有格式化 JSON。
- 历史页会把当前选中记录按以下分组结构化展示：
  - `Meta`
  - `Score`
  - `Media`
  - `Mail`
  - 其他动态字段
- 选中某条历史记录时，会自动与时间上更早的一条记录做比较。
- 当前阶段对比状态先支持三种：
  - `changed`
  - `added`
  - `unchanged`
- 历史列表卡片会直接显示简短变化摘要，如“变化 X 项 / 新增 Y 项 / 未变化 Z 项”。
- 如果当前记录已经是首条历史，则页面明确显示“无上一条可比较”。
- 历史页结构化展示已接入全局字段开关，隐藏字段不会出现在历史详情里。
- 原始 JSON 没有删除，改为折叠在“查看原始 JSON”区域内，作为兜底。

## 12. 本次功能更新

本次更新聚焦“系统配置页字段开关分组化”。

- 系统配置页中的字段开关不再平铺显示路径列表。
- 现在会按以下分组展示：

- 当前列表页摘要仍保留固定区块，但详情页和历史页会真实移除被隐藏的主分组。
