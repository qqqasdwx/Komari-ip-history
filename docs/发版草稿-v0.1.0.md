# v0.1.0 发版草稿

## 版本定位

`v0.1.0` 是 Komari IP History 第一版可追溯 Docker 版本，面向已经使用 Komari 的用户提供 IP 质量历史、快照、独立节点、公开 API、通知和可版本追溯的节点接入能力。

## 主要变化

- 新增 Komari 注入入口，支持默认主题和 PurCarte 主题下查看 IP 质量结果。
- 支持未接入节点显示“去接入”，用户确认后再进入接入配置。
- 支持独立 IPQ 节点，不再要求所有节点必须来自 Komari。
- 节点详情页重构为节点工作区，包含设置、详情、历史和快照。
- 支持多目标 IP、自动发现本机 IP、目标启停、排序和历史记录。
- 支持历史变化事件、快照收藏和快照对比。
- 支持只读 Public API 和 API Key 管理。
- 支持通知系统，包括 Telegram、Webhook 和 JavaScript 发送器。
- 支持上报计划时区、安装后立即执行和重复执行安装命令。
- 接入命令支持版本化安装脚本来源，固定版本镜像会指向同名 tag 的 `deploy/install.sh`。

## 部署建议

正式部署建议使用固定版本镜像：

```bash
ghcr.io/qqqasdwx/komari-ip-history:v0.1.0
```

`latest` 适合快速试用和开发联调，不建议长期用于正式部署。

## 升级注意事项

- 升级前建议备份 SQLite 数据库文件。
- 固定版本镜像启动后，会自动补齐新版本需要的数据库字段。
- 已有节点、目标 IP、历史、快照、通知和 API Key 应保留。
- 如果节点接入脚本来源从旧版本升级到固定版本，建议在节点设置页复制新的接入命令并重新执行一次。
- 重新执行接入命令会替换节点侧 cron 和 reporter 脚本，不会更换节点 token，也不会删除已有历史数据。

## 已知限制

- GitHub Actions 当前不会自动跑完整 Playwright E2E 作为发布门禁，正式发布前仍需要本地执行第十步验收。
- 当前发布产物以 Docker 镜像为主，不提供多平台独立二进制附件。
- 当前不维护 beta、rc 或独立发布分支。

## 真实发版动作清单

真实发版前需要用户再次确认，然后执行：

1. 确认 `dev` 当前提交作为 `v0.1.0` 候选版本。
2. 合并到发布目标分支。
3. 创建 annotated tag：`git tag -a v0.1.0 -m "Release v0.1.0"`。
4. 推送 tag：`git push origin v0.1.0`。
5. 等待 GitHub Actions 构建并发布 `ghcr.io/qqqasdwx/komari-ip-history:v0.1.0`。
6. 创建同名 GitHub Release，并使用本草稿作为 release notes 基础。
