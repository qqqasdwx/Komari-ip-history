## 6. 下一步建议优先级

我认为接下来更值得做的方向是：

1. 继续收口“IP 质量变化”核心能力
   - 决定后续是否要支持更多变化状态
   - 决定字段级规则后续是否需要更强的冲突/覆盖优先级定义
   - 决定是否还需要保留或重做趋势表达

2. 收口阶段 1 剩余体验细节
   - 检查嵌入视图与独立页之间是否还存在行为差异
   - 检查详情页、历史页、变化页是否还能继续减信息量
   - 检查规则配置页是否还需要搜索或折叠能力

3. 补更多真实上报场景约束
   - 错误提示
   - 非法 token 响应
   - 空数据上报处理
   - 生产环境地址配置说明
