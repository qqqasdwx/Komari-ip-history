# 当前工作进度

更新时间：2026-04-02

## 1. 当前状态概览

当前仓库不是从零开始，而是已经具备一套可运行的主链路：

- Go 后端
- React 前端
- 管理员登录
- 节点列表 / 节点详情
- Header 生成
- `embed=1` 弹窗详情页
- Docker 联调环境

但需要注意：

- 代码里仍混有一部分旧方向实现
- 文档和代码直到本轮前还没有完全对齐
- 当前真正需要推进的，已经从“样式整理”转成了“鉴权和接口边界收口”

---

## 2. 当前已经确认的方向

目前已经拍板的关键结论是：

1. 默认按**跨域独立部署**设计
2. `/ipq/` 子路径只保留兼容能力，不再是默认前提
3. 前端主线是 **React**
4. 产品体验上仍然优先争取**页内弹窗**
5. 工程上必须保留**独立页面兜底**
6. 当前重点是**当前 IP 质量结果**
7. “历史变化 / 变化视图 / 趋势分析”不再作为当前主产品方向
8. 开发环境采用：
   - Debian 原生日常开发
   - Docker 联调 / E2E / CI

另外，本轮又补齐了一个新的架构级结论：

- 注入弹窗鉴权以 `docs/注入弹窗鉴权定稿.md` 为准

---

## 3. 当前代码已经具备的能力

### 3.1 前端与部署口径

- 默认前端入口已经切到 React
- 默认部署入口已经切到根路径 `/`
- Header 生成支持独立部署地址
- Compose / CI / Playwright 的默认应用入口也已经从 `/ipq/` 收口到根路径

### 3.2 当前结果展示

- 节点详情页和 `embed=1` 详情页都已经可用
- 详情页整体展示已按 IPQuality 风格收口
- 缺失字段可显示 `N/A`
- 弹窗嵌入页样式也已做过一轮压缩和细修

### 3.3 注入链路基础能力

- loader 可注入节点详情页按钮
- loader 已接入基于 `Komari /api/me` 的管理员 / 游客判断
- 游客只读全局开关已落地
- `iframe` 已拆成：
  - 游客匿名只读弹窗
  - 管理员完整详情弹窗
- 独立页登录和 `connect` 过渡页已接入回跳链路

### 3.4 接口边界

- 管理员详情接口已只返回当前状态
- 历史数据已拆到独立接口
- `/public/...` 匿名只读接口已落地
- 公开接口已只保留展示所需主分组，不再回传内部配置和历史 payload

---

## 4. 当前实现与最新方向之间的主要偏差

### 4.1 当前剩余的主要偏差

这一轮主链路已经落地，`loader` 性能也做完了第一轮收口，而且已经继续清掉了旧前端和变化规则残留，但还有一类问题没有继续收尾：

- 公开只读和管理员完整详情的提示文案还可以继续抠

### 4.2 仍需继续确认的体验细节

当前状态机已经通了，但还有一些体验层问题后面还要继续抠：

- toast 文案是否调整
- 游客只读和管理员完整详情在视觉上是否还要进一步区分
- 管理员回弹窗失败时的独立页落点文案是否需要更明确

### 4.3 loader 性能已做首轮收口

之前审查里提到的 loader 全局扫描 / `MutationObserver` 偏重问题，这一轮已经做了第一轮减法：

- 去掉了全局 `MutationObserver`
- 不再靠整页持续观察触发扫描
- 改成路由变化驱动 + 短时重试
- 去掉了 `document.body.innerText` 这类最重的兜底扫描

当前这块不再是主要阻塞项，后续主要是按具体主题继续做小幅兼容。

---

## 5. 当前对 `connect` 页的定位

代码里现有的 `/#/connect?uuid=...&name=...` 仍然有价值，但它的定位已经变化：

- 它不再应被视为游客和管理员统一共用的唯一主入口
- 更合理的定位是：
  - 管理员登录 / 接入 / 回跳链路中的过渡页

也就是说：

- 这条能力可以保留
- 但其产品地位已经从“总主链路”下降为“管理员支线能力”

---

## 6. 当前更合理的下一步

文档整理到这里已经基本够用，后面重点应转回实现。

更合理的顺序是：

1. 再根据验收反馈继续抠弹窗体验和提示文案
2. 如果后续需要，再按具体主题补 loader 的轻量兼容
3. 最后再考虑是否继续做历史能力的后台收口
