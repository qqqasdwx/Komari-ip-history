# 第四步：自动探查与目标启停

## 目标

让节点侧先向服务端请求上报计划，再按服务端批准的目标 IP 执行探查。支持自动发现 IP 与手动目标 IP 并存，并允许用户按单个目标 IP 启停上报。

## TODO

- 节点侧采集本机可探查的候选 IP。
- 节点侧在探查前向 IPQ 请求上报计划。
- 服务端根据节点配置返回允许探查的目标 IP。
- 自动发现的新 IP 会持久化为目标 IP。
- 自动发现目标带有“自动发现”来源标识。
- 手动添加目标带有“手动添加”来源标识。
- 手动目标即使不在本机候选 IP 中，也能继续被服务端批准。
- 用户可以停用单个目标 IP 的上报。
- 用户可以重新启用单个目标 IP 的上报。
- 停用后的目标 IP 不再接受节点上报。
- 目标 IP 记录最近一次被发现或上报的时间。
- 页面能区分目标来源、启停状态和最近发现时间。

## 不包含

- 不实现通知规则。
- 不实现公开 API。
- 不实现独立节点创建。
- 不实现 Komari 手动绑定。

## 验收清单

- 节点安装脚本会先请求上报计划，再执行探查。
- candidate IP 为空时，仍会请求服务端计划。
- 服务端已有启用的手动目标时，candidate IP 为空也能返回该手动目标。
- 自动发现 IP 会出现在目标列表中。
- 手动 IP 和自动发现 IP 可以同时存在。
- 停用目标后，该目标不再接受上报。
- 重新启用目标后，该目标可以继续上报。
- 页面上能看出每个目标的来源和启停状态。
