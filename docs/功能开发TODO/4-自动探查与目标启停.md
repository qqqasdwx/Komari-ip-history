# 第四步：自动探查与目标启停

## 目标

让节点侧先向服务端请求上报计划，再按服务端批准的目标 IP 执行探查。支持自动发现 IP 与手动目标 IP 并存，并允许用户按单个目标 IP 启停上报。

## TODO

- [x] 节点侧采集本机可探查的候选 IP。
- [x] 节点侧在探查前向 IPQ 请求上报计划。
- [x] 服务端根据节点配置返回允许探查的目标 IP。
- [x] 自动发现的新 IP 会持久化为目标 IP。
- [x] 自动发现目标带有“自动发现”来源标识。
- [x] 手动添加目标带有“手动添加”来源标识。
- [x] 手动目标即使不在本机候选 IP 中，也能继续被服务端批准。
- [x] 用户可以停用单个目标 IP 的上报。
- [x] 用户可以重新启用单个目标 IP 的上报。
- [x] 停用后的目标 IP 不再接受节点上报。
- [x] 目标 IP 记录最近一次被发现时间，并保留最近一次上报时间。
- [x] 页面能区分目标来源、启停状态、最近发现时间和最近上报时间。

- 不实现公开 API。
