## 目标

把节点详情页收敛为只读展示页面，把节点相关的修改配置能力迁移到独立的节点设置页。进入某个节点后，左侧导航从全局菜单切换为节点工作区菜单，降低历史和快照页面的层级。

## 状态

已完成于分支 `feature/5-node-detail-workspace-refactor`，并已合并到 `dev`。

## 用户体验变更

- 进入节点后，左侧导航切换为节点工作区，提供“返回 / 设置 / 详情 / 历史 / 快照”。
- 节点详情页变为只读展示页，配置能力迁移到节点设置页。
- 历史和快照从节点工作区一级入口进入，减少从详情页再跳转的层级。
- 页面内重复的圆角返回按钮已移除，返回能力统一放在侧边栏。
- 节点列表移除“操作”列，绑定来源只展示简洁状态，鼠标移入立即展示绑定的 Komari 节点全名。
- 页面不再展示 UUID 这类内部键值。

## 核心逻辑变更

- 主要变更是前端路由和节点页面结构收敛，未改变后端节点、绑定、上报配置接口的业务语义。
- 旧快照对比地址保留兼容，访问后进入新的节点工作区快照入口。
- 新建独立节点和 Komari “去接入”都进入节点设置页，由同一页面承载接入配置。

- 从节点列表进入任意节点，确认侧边栏显示“返回 / 设置 / 详情 / 历史 / 快照”，页面内没有重复返回按钮。
- 在“详情”页确认只展示当前结果、目标切换、最近更新时间和只读状态，不出现配置控件。
- 在“设置”页确认可以修改节点名称、目标 IP、上报计划、绑定状态，并能查看接入命令。
- 进入“历史”和“快照”页，确认层级清晰、无可见 UUID，旧快照地址不会出现空白页。
- 在节点列表确认无“操作”列，绑定来源 hover 后立即展示 Komari 节点全名。
