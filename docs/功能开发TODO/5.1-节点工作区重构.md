# 第5.1步：节点工作区重构

## 目标

把节点详情页收敛为只读展示页面，把节点相关的修改配置能力迁移到独立的节点设置页。进入某个节点后，左侧导航从全局菜单切换为节点工作区菜单，降低历史和快照页面的层级。

## TODO

- 进入节点后，左侧侧边栏切换为节点工作区导航。
- 节点工作区导航包含“返回”“设置”“详情”“历史”“快照”。
- “返回”回到节点列表。
- “设置”进入节点设置页。
- “详情”进入节点当前结果展示页。
- “历史”进入节点历史记录页。
- “快照”进入节点快照对比页。
- 节点详情页只展示当前结果、目标切换、最近更新时间和只读状态。
- 节点详情页不放修改节点名、绑定、解绑、目标 IP 管理、上报计划、接入命令等配置能力。
- 新增节点设置页，承载节点级配置能力。
- 节点设置页支持修改 IPQ 节点名称。
- 节点设置页展示 Komari 绑定状态。
- 节点设置页支持绑定 Komari 候选节点。
- 节点设置页支持解除 Komari 绑定。
- 节点设置页支持目标 IP 管理。
- 节点设置页支持上报计划设置。
- 节点设置页展示接入命令。
- 节点列表中的“上报设置”入口改为进入节点设置页。
- 新建独立节点后进入节点设置页。
- Komari 入口中的“去接入”进入节点设置页。
- 旧的快照对比 route 保留兼容，并跳转到新的“快照”入口。
- 页面文案统一使用“快照”，不要求用户理解 compare。
- 移除或收口节点列表上的上报设置弹窗，避免配置能力继续堆在弹窗里。

## 不包含

- 不改后端节点、绑定、上报配置接口的业务语义。
- 不重做 IP 质量结果展示视觉设计。
- 不重做历史记录筛选能力。
- 不重做快照对比核心逻辑。
- 不改变默认主题和 PurCarte 注入页的主题适配要求。

## 验收清单

- 进入任意节点后，左侧显示“返回 / 设置 / 详情 / 历史 / 快照”节点工作区导航。
- 节点详情页只展示结果和只读状态，不出现修改配置入口。
- 节点设置页可以修改 IPQ 节点名称。
- 节点设置页可以管理目标 IP 和上报计划。
- 节点设置页可以展示并复制接入命令。
- 节点设置页可以绑定和解除绑定 Komari 节点。
- 历史和快照可以从节点工作区一级导航进入。
- 旧的快照对比地址仍可访问或跳转，不产生空白页。
- Komari “去接入”进入节点设置页，并能完成接入配置。
- 真实 Debian 页面接入链路仍然通过。
- 默认主题和 PurCarte 的嵌入访问流程仍然通过。
