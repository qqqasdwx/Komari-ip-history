# 第七步：通知系统

## 目标

当节点某个目标 IP 的质量字段发生变化时，按用户配置的通知规则发送通知，并提供通道管理、规则管理、测试发送和投递日志。

## TODO

- 管理员可以打开或关闭通知总开关。
- 管理员可以配置通知标题模板。
- 管理员可以配置通知正文模板。
- 管理员可以创建通知通道。
- 通知通道支持 Telegram。
- 通知通道支持 Webhook。
- 通知通道支持 JavaScript sender。
- 管理员可以启用、停用、编辑、删除通知通道。
- 管理员可以对指定通道执行测试发送。
- 管理员可以创建通知规则。
- 通知规则可以按节点范围设置。
- 通知规则可以按目标 IP 范围设置。
- 通知规则可以按字段设置。
- 通知规则可以启用、停用、编辑、删除。
- 当字段从旧值变为新值时触发通知。
- 当字段从无值变为有值时触发通知。
- 当字段从有值变为无值时触发通知。
- 当字段显示文本不变但状态语义变化时也能触发通知。
- 通知内容包含节点、目标 IP、字段、旧值、新值、记录时间。
- 配置了公开访问地址时，通知内容包含详情链接和对比链接。
- 每次投递都记录成功或失败日志。
- 管理员可以查看和清空投递日志。

## 不包含

- 不实现通知去重。
- 不实现复杂告警聚合。
- 不实现用户级多账号权限。
- 不实现第三方通知市场。

## 验收清单

- Telegram 通道可以保存并测试发送。
- Webhook 通道可以保存并测试发送。
- JavaScript sender 可以保存并测试发送。
- 字段变化后，匹配规则会产生投递日志。
- 不匹配规则不会发送通知。
- 禁用通道不会发送通知。
- 禁用规则不会发送通知。
- 删除节点或目标 IP 后，通知规则页面仍能正常打开。
- 通知失败时能在投递日志中看到失败原因。
- JavaScript sender 长时间不返回时，不会卡住整个应用。
