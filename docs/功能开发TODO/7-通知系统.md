# 第七步：通知系统

## 目标

当节点某个目标 IP 的质量字段发生变化时，按用户配置的通知规则发送通知，并提供发送器设置、规则管理、测试发送和投递日志。

## 状态

已完成于分支 `feature/7-notifications`，并已合并到 `dev`。

## 不包含

- 不实现通知去重。
- 不实现复杂告警聚合。
- 不实现多个发送器并存或多通道轮换。
- 不实现通知标题模板。
- 不实现用户级多账号权限。
- 不实现第三方通知市场。

## 验收清单

- Telegram 发送器可以保存消息正文模板并测试发送。
- Webhook 发送器可以保存请求体模板并测试发送。
- JavaScript sender 可以保存并测试发送。
- 字段变化后，匹配规则会产生投递日志。
- 不匹配规则不会发送通知。
- 关闭通知总开关后不会发送通知。
- 禁用规则不会发送通知。
- 删除节点或目标 IP 后，通知规则页面仍能正常打开。
- 通知失败时能在投递日志中看到失败原因。
- JavaScript sender 长时间不返回时，不会卡住整个应用。
