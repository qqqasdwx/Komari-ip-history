# 第一步：基础升级保护

## 状态

已完成于分支 `feature/1-base-upgrade-guard`。

## 目标

为后续功能开发建立安全基线，确保从当前正式数据升级到新功能版本时，已有节点、目标 IP、历史记录、登录用户、设置项都能保留，并且旧入口仍可继续访问。

## TODO

- [x] 保留现有节点数据，不因为新增节点标识、绑定关系或目标 IP 属性而丢失。
- [x] 旧版本已有节点升级后自动补齐新版本需要的基础字段。
- [x] 旧的 Komari 节点标识仍能作为兼容入口访问已有节点。
- [x] 新旧节点访问方式并存，避免用户升级后原有集成入口立即失效。
- [x] 启动应用时能自动修复缺失的基础数据，让旧数据库可以直接升级使用。
- [x] 为后续“独立节点”“Komari 绑定”“自动探查”“通知”等功能提供稳定的数据基础。

## 不包含

- 不实现独立节点页面。
- 不实现 Komari 手动绑定页面。
- 不实现通知系统。
- 不实现公开 API。
- 不调整前端主要交互。

## 验收清单

- [x] 使用旧版本数据库启动新版本，应用能正常启动。
- [x] 旧数据库中已有多个节点时，升级不失败。
- [x] 升级后旧节点仍能在节点列表中显示。
- [x] 旧节点详情、历史记录、目标 IP 仍能访问。
- [x] 旧 Komari 集成入口仍能定位到对应节点。
- [x] 新增的基础字段对用户不可见或不打扰现有流程。
