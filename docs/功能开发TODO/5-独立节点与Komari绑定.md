# 第五步：独立节点与 Komari 绑定

## 目标

把 IPQ 节点作为主实体，Komari 节点只是可选绑定来源。支持用户创建不依赖 Komari 的独立节点，也支持后续手动绑定或解绑 Komari 节点。

## 状态

已完成于分支 `feature/5-independent-node-binding`，并已合并到 `dev`。

## 用户体验变更

- 节点列表支持直接创建不依赖 Komari 的独立节点。
- 节点设置页展示绑定状态，支持修改节点名称、选择 Komari 候选节点、解除绑定。
- 绑定候选会展示可用和已占用状态，避免用户误把一个 Komari 节点重复绑定到多个 IPQ 节点。
- 解除绑定后，节点不会被删除，会回到独立节点状态继续保留目标 IP、上报计划和历史数据。

## 核心逻辑变更

- IPQ 节点成为主实体，Komari 节点只作为可选外部来源绑定。
- 从 Komari 入口产生的待接入节点可以作为后续手动绑定候选。
- 绑定关系 enforce 一对一约束：一个 IPQ 节点最多绑定一个 Komari 节点，一个 Komari 节点最多被一个 IPQ 节点绑定。

## 手工验收建议

- 在 IPQ 节点列表点击“新建节点”，确认新节点进入设置页，且显示为独立节点。
- 添加目标 IP 后确认接入命令可生成，节点详情页只显示只读结果，不显示配置控件。
- 创建或使用一个 Komari 待接入节点，确认可以在绑定候选中选择并完成绑定。
- 再创建第二个 IPQ 节点，确认同一个 Komari 候选显示为已占用，不能静默重复绑定。
- 对已绑定节点执行解除绑定，确认节点仍保留为独立节点。
