# 第五步：独立节点与 Komari 绑定

## 目标

把 IPQ 节点作为主实体，Komari 节点只是可选绑定来源。支持用户创建不依赖 Komari 的独立节点，也支持后续手动绑定或解绑 Komari 节点。

## TODO

- 用户可以在 IPQ 后台创建独立节点。
- 独立节点可以配置目标 IP、上报计划和安装脚本。
- 节点列表能区分“独立节点”和“Komari 绑定节点”。
- 用户可以修改 IPQ 节点名称。
- 用户可以查看当前节点是否绑定 Komari。
- 用户可以从候选列表中选择 Komari 节点进行绑定。
- 一个 IPQ 节点最多绑定一个 Komari 节点。
- 一个 Komari 节点最多绑定一个 IPQ 节点。
- 用户可以解除 Komari 绑定。
- 解除绑定后，IPQ 节点仍然保留为独立节点。
- 从 Komari 入口自动产生的待接入节点，可以在后台被识别为绑定候选。
- 绑定候选列表能展示 Komari 节点名称和占用状态。

## 不包含

- 不实现通知系统。
- 不实现公开 API。
- 不实现双环境验收脚本。
- 不重做节点详情视觉设计。

## 验收清单

- 可以创建一个没有 Komari 绑定的独立节点。
- 独立节点能添加目标 IP 并生成上报配置。
- 可以把独立节点绑定到一个 Komari 节点。
- 已被其它节点绑定的 Komari 节点不能被静默重复绑定。
- 可以解除绑定，解除后节点不丢失。
- Komari 自动发现的待接入节点能作为绑定候选出现。
- 节点列表和详情页都能明确展示绑定状态。
