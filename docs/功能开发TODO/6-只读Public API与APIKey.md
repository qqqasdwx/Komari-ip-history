# 第六步：只读 Public API 与 API Key

## 目标

提供稳定的只读对外 API，让外部系统可以用 API Key 查询节点、目标 IP、当前结果和历史变化，但不开放写操作。

## TODO

- 管理员可以创建 API Key。
- API Key 明文只在创建时展示一次。
- 管理员可以查看 API Key 列表。
- 管理员可以启用、停用、删除 API Key。
- API Key 调用公开 API 时需要带认证头。
- 未带 API Key 或 API Key 无效时返回拒绝访问。
- 停用的 API Key 不能继续访问。
- 公开 API 支持查询节点列表。
- 公开 API 支持查询节点详情。
- 公开 API 支持查询某节点的目标 IP 列表。
- 公开 API 支持查询某个目标 IP 当前详情。
- 公开 API 支持查询历史记录。
- 公开 API 支持查询历史变化事件。
- 公开 API 支持基础分页和日期范围筛选。
- 管理员可以查看 API 访问日志。
- API Key 访问有基础限流保护。

## 不包含

- 不开放节点创建、修改、删除。
- 不开放目标 IP 写操作。
- 不开放通知配置写操作。
- 不承诺第三方 SDK。

## 验收清单

- 创建 API Key 后，只展示一次明文 key。
- 使用有效 API Key 可以查询节点列表。
- 使用无效 API Key 会被拒绝。
- 停用 API Key 后访问被拒绝。
- 删除 API Key 后访问被拒绝。
- 节点详情、目标 IP、历史记录、历史事件接口都能返回数据。
- 分页、日期范围和字段筛选行为可验证。
- API 访问日志能记录 key、路径、状态码和访问时间。
- 连续高频请求会触发限流。
