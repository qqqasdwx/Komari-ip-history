## 用户体验变更

- Komari 节点页点击 IPQ 入口时，弹窗只用于查看当前 IP 质量。
- Komari 注入入口会先显示“检测接入状态”，状态确认后已接入显示“查看 IP 质量”，未接入显示“开启 IP 质量检测”。
- 默认主题首页节点卡片新增 IPQ 图标入口；入口挂在节点名右侧的原生图标组里，复用默认主题 `IconButton` 风格，已接入使用强调色，未接入使用灰色。
- PurCarte 主题下的未接入提示页和已有数据查看弹窗都复用 PurCarte 毛玻璃背景，不显示默认主题的纯白面板。
- PurCarte 主题下，用户已登录 Komari 但未登录 IPQ 时，注入弹窗会保留在 Komari 页面内显示“登录 IPQ 后查看”按钮，不自动跳到 IPQ 白色登录页。
- 已有目标 IP 的节点会在 Komari 弹窗中打开当前质量结果。
- 没有目标 IP 的节点会先显示“去接入”按钮，不会立即创建 IPQ 节点。
- 用户点击“去接入”后，才会在 IPQ 中创建或更新待配置节点，并在新标签页打开“节点列表 + 上报设置弹窗”。
- 从 Komari 打开的上报设置弹窗会提示用户配置后返回 Komari 重新查看。
- 配置保存后不再自动跳回 Komari，也不会自动重新打开 Komari 弹窗。
- 未登录访客在游客只读关闭时只看到禁止访问提示；游客只读开启时可以查看公开当前结果。

## 核心逻辑变更

- Komari loader 会先只读查询节点接入状态，再决定打开查看弹窗或展示“去接入”按钮。
- Komari loader 会缓存短时间的接入状态，并在页面重新获得焦点时刷新，避免配置后回到 Komari 仍显示旧状态。
- IPQ 新增 Komari 入口只读状态查询和显式接入能力；只有显式接入动作会创建待配置节点壳子。
- 节点详情、上报设置和安装配置继续兼容旧 Komari UUID，同时优先使用 IPQ 自有 `node_uuid`。

## 手工验收建议

- 默认主题 Komari：点击一个已有目标 IP 的节点，确认只出现当前质量弹窗。
- 默认主题 Komari 首页：确认每个节点卡片都有 IPQ 图标；图标位于节点名右侧原有图标组，不出现在节点名前；已有目标 IP 的节点标题为“查看 IP 质量”且为强调色，未接入节点标题为“开启 IP 质量检测”且为灰色。
- 默认主题 Komari：点击一个没有目标 IP 的节点，确认只出现“去接入”按钮，且 IPQ 节点不会在按钮点击前创建。
- 默认主题 Komari：点击“去接入”后，确认新标签页打开 IPQ 节点列表并自动弹出上报设置。
- PurCarte 主题 Komari：用名称包含 `PurCarte` 的测试节点重复上述已配置、未配置确认和“去接入”三条路径，并确认未接入提示页、已有数据查看弹窗都是毛玻璃背景，不是纯白面板。
- PurCarte 主题 Komari：在未登录 IPQ 的浏览器上下文中点击已有数据节点，确认仍停留在 Komari 弹窗内，并显示毛玻璃风格的“登录 IPQ 后查看”按钮。
- 在 IPQ 上报设置中添加目标 IP 后，确认页面提示返回 Komari；回到 Komari 重新点击 IPQ 后查看当前结果。
- 退出登录后分别打开和关闭“允许游客查看已接入节点的当前结果”，确认游客可读和禁止访问两种状态。
