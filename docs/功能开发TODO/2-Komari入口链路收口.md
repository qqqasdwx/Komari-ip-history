# 第二步：Komari 入口链路收口

## 目标

修正 Komari 中点击 IPQ 图标后的职责边界：Komari 弹窗只用于查看当前 IP 质量，接入、配置、登录和管理流程统一进入 IPQ 独立后台。

## TODO

- 已接入节点点击图标时，在 Komari 弹窗中展示当前 IP 质量。
- 未接入节点点击图标时，不再把后台配置页塞进 Komari 弹窗。
- 未接入节点点击图标后，自动在 IPQ 中创建或更新一个待接入节点壳子。
- 未接入节点点击图标后，新标签页打开 IPQ 节点列表页。
- IPQ 节点列表页根据入口参数自动打开对应节点的上报设置弹窗。
- 配置完成后给出明确提示，让用户返回 Komari 重新查看。
- 访客读权限开启时，未登录用户可以按权限查看公开的当前质量结果。
- 访客读权限关闭时，未登录用户看到明确的禁止访问状态。

## 不包含

- 不实现独立节点创建。
- 不实现 Komari 手动绑定。
- 不实现自动探查计划。
- 不实现通知系统。
- 不新增公开 API。

## 验收清单

- 默认主题 Komari 中，已接入节点弹窗只显示查看内容。
- 默认主题 Komari 中，未接入节点会打开 IPQ 独立后台并自动弹出配置。
- PurCarte 主题 Komari 中，同样满足已接入和未接入行为。
- 登录态、未登录访客可读、未登录访客禁用三种状态都有明确结果。
- 完成配置后不会自动重新打开 Komari 弹窗。
- 原有 embed 查看能力不回退。
