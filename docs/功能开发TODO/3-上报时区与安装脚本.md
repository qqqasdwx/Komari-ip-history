# 第三步：上报时区与安装脚本

## 目标

让节点上报计划的 cron 执行时区由 IPQ 明确管理，避免节点机器本地时区不同导致执行时间不符合用户预期。

## TODO

- 节点上报设置中保存 cron 表达式。
- 节点上报设置中保存 cron 解析时区。
- 默认时区优先使用用户浏览器当前时区。
- 用户可以手动选择或修改执行时区。
- 页面明确展示当前 cron 按哪个时区解析。
- 页面展示后续执行时间预览。
- 安装配置中包含 cron、时区、是否立即执行等信息。
- 生成的安装脚本写入明确的 cron 时区。
- 用户修改上报时间或时区后，可以重新安装或更新节点侧任务。
- 保留立即执行一次的开关。

## 不包含

- 不实现自动探查目标 IP。
- 不实现目标 IP 启停。
- 不实现通知系统。
- 不实现独立节点绑定。

## 验收清单

- 新节点打开上报设置时能看到默认时区。
- 保存时区后再次打开页面，时区保持不变。
- 修改 cron 后能看到下一次执行时间预览。
- 修改时区后预览结果按新时区变化。
- 安装脚本中能看到明确的时区配置。
- 立即执行开关能保存并反映到安装配置中。
- 无效 cron 和无效时区都有清晰错误提示。
