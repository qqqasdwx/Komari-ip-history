# 第三步：上报时区与安装脚本

## 目标

让节点上报计划的 cron 执行时区由 IPQ 明确管理，避免节点机器本地时区不同导致执行时间不符合用户预期。

## 已完成说明

- 后端为节点上报配置增加 `schedule_timezone`，保存到数据库并参与 cron 预览、安装配置和安装脚本生成。
- 前端上报设置弹窗增加“解析时区”选择，未保存过时区的新节点默认使用浏览器时区，并展示“当前 Cron 按某时区解析”和最近 10 次执行时间。
- 安装配置 JSON 返回 `schedule_cron`、`schedule_timezone`、`run_immediately`；安装脚本写入 `CRON_TZ` 和 `TZ`，重新运行安装命令会替换节点侧 cron 配置。

## 手工验收建议

- 打开节点列表的“上报设置”，确认“解析时区”、当前解析时区提示和最近 10 次执行时间符合预期。
- 将 Cron 和时区改成一个容易判断的组合，例如 `*/30 * * * *` + `Asia/Shanghai`，关闭后重新打开确认配置保留。
- 复制接入命令并在测试节点执行，确认 `/etc/cron.d/ipq-reporter-*` 中包含 `CRON_TZ=Asia/Shanghai` 和 `TZ=Asia/Shanghai`。
