# 项目背景

## 1. 项目是什么

这是一个与 `Komari` 集成的“节点 IP 质量展示服务”。

它不是 `Komari` 官方源码的一部分，而是一个**独立部署的服务**，主要负责：

- 接收或存储节点的 IP 质量结果
- 提供后台管理页面
- 提供节点详情展示页面
- 生成可复制到 `Komari` 的 header

---

## 2. 想解决什么问题

项目的核心诉求不是单独做一个新面板，而是：

> 用户已经在 `Komari` 里管理节点，希望在 `Komari` 节点详情页里顺手看到该节点的 IP 质量结果。

也就是说，主要体验仍然希望发生在 `Komari` 内，而不是要求用户长期单独打开本服务页面。

---

## 3. 当前产品目标

当前产品目标可以概括为：

1. 用户独立部署本服务
2. 在本服务后台中生成一段 `Komari header`
3. 把这段 header 粘贴到 `Komari`
4. 在 `Komari` 节点详情页里出现入口按钮
5. 用户点击后，优先以**页内弹窗**方式查看当前 IP 质量
6. 如果弹窗受限，则退回**独立页面**查看

---

## 4. 当前集成原则

- 不修改 `Komari` 官方源码
- 通过 `Komari` 自定义 header 注入少量 JavaScript 实现集成
- 注入脚本尽量薄，只做桥接、挂载和入口注入
- 复杂样式和展示逻辑尽量由本服务承担
- 默认按**跨域独立部署**考虑，不再把“同域 `/ipq/` 子路径”当成前提

---

## 5. 当前展示重点

当前最重要的展示目标是：

- **当前 IP 质量结果**

页面风格上：

- 后台整体尽量贴近 `Komari`
- 节点结果区尽量贴近 `IPQuality` 原始报告观感

不再把“历史变化 / 变化视图 / 趋势分析”作为当前主产品方向。

---

## 6. 当前技术方向

- 后端：Go
- 前端：独立 Web 前端
- 数据库：SQLite
- 数据结构参考：`references/IPQuality/`
