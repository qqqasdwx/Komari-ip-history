# 代码影响清单

本文档用于根据当前已经确认的新方向，梳理现有代码库里**会受影响的模块、文件和实现假设**。

当前判断基于以下最新口径：

- 默认按**跨域独立部署**设计
- Header 只做**桥接和入口注入**
- 产品体验上仍然**优先争取页内弹窗**
- 工程上必须保留**独立页面兜底**
- 注入弹窗鉴权以 `docs/注入弹窗鉴权定稿.md` 为准
- 当前重点是**当前 IP 质量结果**

---

## 1. 当前最主要的代码影响面

以下四条主线目前都已经落了一版代码，文档保留它们主要是为了指导下一轮继续收尾：

从代码结构看，当前受影响最大的不是某一个函数，而是下面四条线：

1. **权限与公开接口线**
   - 需要把：

     - 管理员完整链路

3. **详情 / 历史接口边界线**

4. **loader 与遗留实现收尾线**
   - iframe 弹窗骨架可以继续保留
   - 但 loader 扫描策略、legacy 残留和旧 `connect` 叙事都需要收口

## 2. 当前最重要的风险点

### 2.1 legacy 噪音仍在

`loader` 这一块已经做过一轮减法，当前更明显的问题反而是 legacy 和旧变化线残留。它们会持续增加：

- 理解成本
- 回归范围
- 性能风险

### 2.2 匿名只读接口目前仍信任注入链路的前端分流

当前游客公开边界已经可用，但仍然主要依赖：

- `Komari /api/me`
- `Komari /api/nodes`

这意味着：

- 公开接口本身不再重复校验 `Komari` 节点公开状态
- 公开边界当前主要靠注入链路保证

这个边界是当前已接受的产品取舍，但后续要继续留意。
