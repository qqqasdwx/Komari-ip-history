# 代码影响清单

本文档用于根据当前已经确认的新方向，梳理现有代码库里**会受影响的模块、文件和实现假设**。

当前判断基于以下最新口径：

- 默认按**跨域独立部署**设计
- Header 只做**桥接和入口注入**
- 产品体验上仍然**优先争取页内弹窗**
- 工程上必须保留**独立页面兜底**
- 注入弹窗鉴权以 `docs/注入弹窗鉴权定稿.md` 为准
- 当前重点是**当前 IP 质量结果**

---

## 1. 当前最主要的代码影响面

从代码结构看，当前受影响最大的不是某一个函数，而是下面四条线：

1. **权限与公开接口线**
   - 需要把：
     - 游客只读
     - 管理员完整链路
     真正拆开

2. **弹窗状态机与回跳线**
   - `Komari` 游客和管理员的点击行为已经拍板
   - 但代码里还没有按这个状态机落地

3. **详情 / 历史接口边界线**
   - 当前详情接口仍然默认带历史
   - 已不符合“详情只看当前状态”的最新结论

4. **loader 与遗留实现收尾线**
   - iframe 弹窗骨架可以继续保留
   - 但 loader 扫描策略、legacy 残留和旧 `connect` 叙事都需要收口

---

## 2. 当前最重要的风险点

### 2.1 鉴权模型仍然是旧实现导向

当前代码主链路还明显带着旧假设：

- 试图在嵌入链路里完成更多登录动作
- `connect` 页承担了过多统一入口职责
- 游客与管理员链路没有真正分开

但现在已经拍板：

- 游客只读走匿名公开接口
- 管理员链路走独立页登录 / 接入 / 回跳
- 不再把“跨站 iframe 里复用管理员 cookie”作为主设计

### 2.2 详情接口与历史接口仍未拆开

当前详情接口仍会把历史数据一起带出来。

这会直接影响：

- 嵌入弹窗打开速度
- 详情接口负载
- 游客只读接口的最小返回结构

### 2.3 loader 还没有按新状态机收口

当前 loader 已经能：

- 注入按钮
- 打开弹窗
- 打开独立页

但它还没有真正按最新状态机分流：

- `Komari` 游客 + 游客只读关闭 -> toast
- `Komari` 游客 + 游客只读开启 -> 匿名只读弹窗
- `Komari` 管理员 + IPQ 未登录 -> 跳独立页登录并继续原操作
- `Komari` 管理员 + 未接入 -> 跳独立页接入并优先回弹窗

### 2.4 loader 性能和 legacy 噪音仍在

这不再是最高优先级问题，但仍会持续增加：

- 理解成本
- 回归范围
- 性能风险

## 3. P0：必须先收口的代码影响

### 3.1 Komari 身份判断与弹窗状态机

对应文件：

- `internal/service/header.go`
- `internal/httpx/handlers/embed.go`
- `internal/httpx/handlers/admin.go`
- `internal/httpx/handlers/auth.go`
- `web/src/react-preview/App.tsx`

当前问题：

1. 还没有接入基于 `Komari` 接口的身份判断
2. 还没有游客只读总开关落点
3. 还没有匿名只读公开接口
4. 还没有把 `toast / 只读弹窗 / 独立登录 / 自动接入 / 回弹窗` 串成确定状态机
5. `connect` 页仍然更像旧主链路产物，而不是最新管理员支线能力

建议动作：

- 保留现有 loader 骨架
- 直接开始把它改造成新状态机
- 不再继续围绕“跨站 iframe 登录闭环”补丁式修修补补

### 3.2 详情接口与历史接口拆分

对应文件：

- `internal/service/nodes.go`
- `internal/httpx/handlers/nodes.go`
- `internal/httpx/handlers/embed.go`
- `web/src/react-preview/App.tsx`

当前问题：

1. 详情接口负载与当前产品目标不一致
2. 游客只读接口后续很难直接复用当前详情结构
3. 历史能力虽然还可以保留，但不应继续绑在详情接口里

建议动作：

- 尽快把详情和历史拆开
- 管理员详情接口与游客只读接口都只返回当前状态
- 历史保留为独立接口和后台能力

### 3.3 游客只读链路与公开边界

对应文件：

- `internal/httpx/handlers/embed.go`
- `internal/httpx/handlers/admin.go`
- `internal/service/header.go`
- `web/src/react-preview/App.tsx`

当前问题：

1. 还没有游客只读全局开关
2. 还没有 `/public/...` 公开只读接口
3. 还没有按 `Komari` 的节点公开状态和 IP 游客公开策略做裁剪

建议动作：

- 单独增加游客只读公开接口
- IP 展示跟随 `Komari` 的游客公开策略
- 非 IP 的当前结果可在游客只读中完整展示

## 4. P1：应该继续做减法的部分

### 4.1 history / changes / 变化规则 残留

对应文件：

- `internal/service/nodes.go`
- `internal/service/settings.go`
- `internal/httpx/handlers/nodes.go`
- `web/src/legacy-app.ts`
- `web/src/react-preview/lib/changes.ts`

当前判断：

- 历史能力可以保留
- 但不应再默认绑在当前详情页上
- 变化规则和旧变化视图残留应继续退出主线

### 4.2 legacy 前端残留

对应文件：

- `web/src/legacy-app.ts`
- `web/src/main.ts`

当前判断：

- React 已是唯一主线
- legacy 继续保留只会增加维护噪音
- 但这不是比鉴权和接口拆分更优先的问题
