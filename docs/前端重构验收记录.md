阶段目标：记录当前 React SPA 的页面状态，作为后续 Tailwind 与 shadcn/ui 重构的视觉和行为对比基线。

本阶段 UI 改动：无。仅新增可复用 Playwright 基线截图脚本。

测试数据：

- 通过开发种子数据覆盖节点列表、无数据节点、多 IP 历史、多快照对比和长名称密集数据。
- 本轮截图使用 `开发种子-多IP历史` 和 `开发种子-多快照对比`。
- 可用 `docker compose -f compose.dev.yml exec -T workspace sh /workspace/deploy/dev/workspace/seed-dev-nodes.sh` 重新生成。

## 02. 引入 UI 基础设施

阶段目标：加入 shadcn/ui 风格基础设施，先不替换现有页面，降低后续页面重构风险。

本阶段改动：

- 新增 `web/src/react-preview/lib/utils.ts`，提供 `cn()`。
- 新增基础组件：`Button`、`Input`、`Label`、`Card`、`Badge`。
- 新增依赖：`class-variance-authority`、`clsx`、`tailwind-merge`、`@radix-ui/react-slot`、`@radix-ui/react-label`、`lucide-react`。
- 修复 E2E 过程中发现的 Komari 登录返回弹窗时序问题：通过 resume 恢复的弹窗会忽略滞后的 `open-standalone` 消息，避免 overlay 被误关闭。
- 页面行为和现有路由不变。

执行命令：

```bash
docker compose -f compose.dev.yml exec -T workspace sh -lc 'cd /workspace/web && npm run build'
cd web && node --check playwright/capture-frontend-baseline.mjs
docker compose -f compose.dev.yml exec -T workspace sh -lc 'cd /workspace/web && FRONTEND_REFACTOR_STAGE=02-ui-infra IPQ_PUBLIC_BASE_URL=http://127.0.0.1:8090 node playwright/capture-frontend-baseline.mjs'
docker compose -f compose.dev.yml exec -T workspace go test ./...
docker compose -f compose.dev.yml exec -T workspace sh /workspace/deploy/dev/workspace/run-e2e.sh
```
