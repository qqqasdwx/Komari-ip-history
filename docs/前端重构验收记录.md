# 前端重构验收记录

## 01. 建立基线

阶段目标：记录当前 React SPA 的页面状态，作为后续 Tailwind 与 shadcn/ui 重构的视觉和行为对比基线。

本阶段 UI 改动：无。仅新增可复用 Playwright 基线截图脚本。

测试数据：

- 通过开发种子数据覆盖节点列表、无数据节点、多 IP 历史、多快照对比和长名称密集数据。
- 本轮截图使用 `开发种子-多IP历史` 和 `开发种子-多快照对比`。
- 可用 `docker compose -f compose.dev.yml exec -T workspace sh /workspace/deploy/dev/workspace/seed-dev-nodes.sh` 重新生成。

执行命令：

```bash
cd web && node --check playwright/capture-frontend-baseline.mjs
docker compose -f compose.dev.yml exec -T workspace sh -lc 'cd /workspace/web && IPQ_PUBLIC_BASE_URL=http://127.0.0.1:8090 node playwright/capture-frontend-baseline.mjs'
```

人工检查结果：

- 所有截图均已查看，页面不是 loading、空白页、错误页或骨架屏。
- 节点列表同时覆盖有数据和无数据节点；移动端能展示完整节点卡片。
- 详情页、历史页和对比页均显示真实种子报告和历史变化。
- 接入配置页显示当前 `loader` 和完整内联版 Header 内容。

遗留风险：

- 本阶段只建立基线，不改变页面结构。
- 公开访问和 Komari 默认/PurCarte 注入截图由 `run-e2e.sh` 继续覆盖，后续嵌入相关重构阶段再写入专门对比记录。
