# IPQ 用户操作手册

- 演示环境：Komari `http://127.0.0.1:8080` · IPQ `http://127.0.0.1:8090`
- 演示节点：`IPQ 手册演示节点`
- 演示目标 IP：`1.1.1.1`、`8.8.8.8`

## 说明

- 本手册的截图全部来自真实点击流程，不是手工拼图。
- 覆盖范围包括：接入配置、Komari 注入、多 IP 上报、当前详情、历史记录、快照对比、收藏快照、游客只读、历史保留、用户设置，以及默认主题和 PurCarte 主题下的注入体验。
- 接入命令在演示过程中按页面生成内容原样执行，节点侧没有手工补参数。

## 操作步骤

### 1. Komari 初始空白页

- 目的：从空数据开始演示，方便看清节点是如何一步步接入到 IPQ 的。
- 操作：在空数据状态下打开 Komari 首页。
- 结果：通过

![Komari 初始空白页](assets/用户操作手册/01-komari-empty.png)

### 2. 登录 Komari 管理后台

- 目的：先登录 Komari，后续才能创建节点、写入自定义 Header 和切换主题。
- 操作：在 Komari 首页点击登录，输入默认账号 `admin / admin`。
- 结果：通过

![登录 Komari 管理后台](assets/用户操作手册/02-komari-login.png)

### 3. 进入 Komari 后台首页

- 目的：确认已经进入 Komari 管理工作区。
- 操作：提交登录表单后进入 `/admin`。
- 结果：通过

![进入 Komari 后台首页](assets/用户操作手册/03-komari-admin-home.png)

### 4. 创建 Komari 节点

- 目的：IPQ 依赖 Komari 节点 UUID 建立映射，所以要先在 Komari 中创建节点。
- 操作：在 Komari 节点管理页点击 Add，填写节点名称“IPQ 手册演示节点”。
- 结果：通过

![创建 Komari 节点](assets/用户操作手册/04-komari-add-node.png)

### 5. 节点创建完成

- 目的：确认演示节点已经出现在 Komari 列表中。
- 操作：提交创建表单后，节点出现在 Komari 首页列表。
- 结果：通过

![节点创建完成](assets/用户操作手册/05-komari-node-created.png)

### 6. 登录 IPQ 后台

- 目的：进入 IPQ 管理后台，后续配置接入代码和节点上报。
- 操作：在 IPQ 登录页输入默认账号 `admin / admin`。
- 结果：通过

![登录 IPQ 后台](assets/用户操作手册/06-ipq-login.png)

### 7. IPQ 初始节点列表为空

- 目的：在节点真正接入之前，IPQ 节点页不会出现任何数据。
- 操作：登录 IPQ 后打开节点列表。
- 结果：通过

![IPQ 初始节点列表为空](assets/用户操作手册/07-ipq-empty-nodes.png)

### 8. 查看接入配置页

- 目的：接入配置页集中提供接入地址、游客只读、loader 版 Header 和完整内联版 Header。
- 操作：打开 IPQ 的“接入配置”页面。
- 结果：通过

![查看接入配置页](assets/用户操作手册/08-integration-loader.png)

### 9. 查看完整内联版代码

- 目的：如果不想依赖 loader，也可以使用完整内联版 Header。
- 操作：滚动到“完整内联版”区域，查看备用的完整内联代码。
- 结果：通过

![查看完整内联版代码](assets/用户操作手册/09-integration-inline.png)

### 10. 将 Header 注入到 Komari

- 目的：只需要把 IPQ 生成的 Header 粘贴到 Komari 的 Custom Header 中，就能启用注入按钮。
- 操作：进入 Komari 的 Site 设置，把 loader 版代码粘贴到 `Custom Header`。
- 结果：通过

![将 Header 注入到 Komari](assets/用户操作手册/10-komari-custom-header-filled.png)

### 11. 保存 Komari Header

- 目的：保存之后，Komari 页面会自动加载 IPQ 注入层。
- 操作：点击 Custom Header 区域下方的 Save。
- 结果：通过

![保存 Komari Header](assets/用户操作手册/11-komari-custom-header-saved.png)

### 12. 在 Komari 节点详情中看到注入入口

- 目的：注入成功后，节点详情页会出现“IPQ”入口，管理员可以直接从 Komari 内操作。
- 操作：打开刚创建的 Komari 节点详情页，确认注入按钮已出现。
- 结果：通过

![在 Komari 节点详情中看到注入入口](assets/用户操作手册/12-komari-detail-injected-button.png)

### 13. 通过注入入口接入节点

- 目的：未接入的节点会先走独立接入页面，接入完成后就能继续在 Komari 弹窗中查看。
- 操作：点击注入按钮，完成节点接入。
- 结果：通过

![通过注入入口接入节点](assets/用户操作手册/13-komari-connect-result.png)

### 14. 节点出现在 IPQ 列表中

- 目的：节点接入后，会出现在 IPQ 的节点列表中，状态初始为“无数据”。
- 操作：回到 IPQ 节点列表，确认刚接入的节点已经出现。
- 结果：通过

![节点出现在 IPQ 列表中](assets/用户操作手册/14-ipq-node-list-connected.png)

### 15. 打开上报设置

- 目的：上报设置弹窗用于管理目标 IP、执行周期和接入命令。
- 操作：点击节点行内的齿轮图标，打开“节点上报设置”。
- 结果：通过

![打开上报设置](assets/用户操作手册/15-report-config-empty.png)

### 16. 配置多个目标 IP 和执行计划

- 目的：一个节点可以维护多个目标 IP。Cron 和“安装后立即执行一次”会实时保存，底部接入命令也会随配置变化。
- 操作：在上报设置里添加 `1.1.1.1`、`8.8.8.8` 两个目标 IP，并把 Cron 调整为 `*/30 * * * *`。
- 结果：通过

![配置多个目标 IP 和执行计划](assets/用户操作手册/16-report-config-multi-ip.png)

### 17. 复制接入命令

- 目的：接入命令会根据当前节点配置动态生成，重复执行同一条命令只会覆盖更新本地配置。
- 操作：点击接入命令右侧的“复制”。
- 结果：通过

![复制接入命令](assets/用户操作手册/17-report-config-command-copied.png)

### 18. 节点列表显示有数据

- 目的：节点开始上报后，列表状态会从“无数据”变成“有数据”，最近更新时间也会刷新。
- 操作：等待节点侧命令执行完成，再回到 IPQ 节点列表查看结果。
- 结果：通过

![节点列表显示有数据](assets/用户操作手册/18-node-list-after-report.png)

## 补充说明

- `loader 版` 是推荐方案，后续更新注入逻辑时一般不需要重新复制整段 Header。
- 节点上报设置里的接入命令会跟随目标 IP、Cron 和立即执行开关实时变化。
- 历史页主要看“字段变化”，快照对比页主要看“某两个时间点的完整结果差异”。
- 游客只读只影响 Komari 注入弹窗，不影响 IPQ 后台管理员视图。
- 历史保留是全局设置，收藏快照不会被自动清理。
- 快照对比至少需要两条历史快照；如果节点刚接入、暂时只有一条快照，页面会先提示继续等待后续上报。
